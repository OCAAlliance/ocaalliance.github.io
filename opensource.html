Open-Source Software

This page is a quick guide for users of and contributors to OCA Alliance open-source software.

Alliance open-source products

An Alliance open-source product is a software component or system the Alliance publishes to help companies use AES70, the Audio Engineering Society's OCA standard for audio device control and monitoring over networks.

Going forward, the Alliance will offer a growing number of open-source products to assist AES70 evaluation, experimentation, education, demonstration, and commercial implementation.

Finding Alliance open-source products

All Alliance open-source products are listed in various entries of the Downloads section of this website.

Physically, Alliance open-source products are in GitHub repositories that reside in subfolders of the OCA Alliance's Github site, ocaalliance.github.io.

Using Alliance open-source products

Alliance open-source products are licensed under the Apache 2.0 license . You may use reproduce, modify, and/or redistribute any Allliance open-source product according to the terms of this license.

For complete licensing details, please go to the Licensing section of this website.

Contributing to Alliance open-source projects

Contributions to Alliance open-source projects are welcome. To make a contribution, the contributor posts a git pull-request to the product's GitHub repository. Once the repository manager approves it, the pull-request will be merged into the product.

To become a contributor, an individual must first sign an Individual Contributor License Agreement and submit it to the Alliance. Once the individual's submission is accepted, they can contribute to any Alliance open-source project.

In addition, a company whose employees contribute corporate intellectual property to an Alliance open-source project should sign a Corporate Contributor License Agreement and submit it to the Alliance. Once a company's CCLA is accepted, its employees can contribute to any Alliance open-source project.

Detailed procedures for signing and submitting Contributor License agreements are in the Licensing section of this website, here

Technical Details

Repository structure

All Alliance repositories have the following base folder set: 

Folder or File		Contents		
[component 1]/		Component (1) code ...
[component n]/		Component (n) code
License/		copy of Apache 2.0 license
Documents/		Product documentation 
README.md		Readme file - see next.

README file
